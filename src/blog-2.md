Object-oriented programming (OOP) organizes code into objects and classes. In large-scale TypeScript projects, OOP helps developers write modular, reusable, scalable, and maintainable code.

The four main pillars of OOP are:

1. Inheritance

Inheritance allows a class to inherit properties and methods from another class.
It allows developers to reuse existing logic instead of rewriting code over and over again.

2. Polymorphism

Polymorphism means “many forms.”

Flexible architecture.
Different classes can behave differently through the same interface.
Esy extension.
New features can be added without changing existing code.
Clean logic.

3. Abstraction
   Abstraction hides the details of the internal implementation and exposes only the essential functionality.
   It helps to focus on what an object does instead of how it works internally, thereby reducing complexity.

4. Encapsulation
   Encapsulation protects data by restricting direct access to internal properties.

It uses:
Private
Protected
Getters/Setters
To control data access.

Data protection.
Prevents accidental changes.
Improved security.
Sensitive logic is hidden.
